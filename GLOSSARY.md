# Glossary

This glossary names the stable terms used across the theatron repo.
The machine-readable companion is [`_llm/glossary.toml`](./_llm/glossary.toml).

| Term | Meaning |
|---|---|
| `theatron` | Place for viewing. The fleet desktop UI infrastructure workspace. |
| `themelion` | Foundation. Theme provider and UI foundation crate. |
| `mekhane` | Stage machinery. Windowing, event loop, tray, menu, and hotkey crate. |
| `skeue` | Equipment or props. Generic Dioxus component crate. |
| `gramma` | Written character. Markdown, syntax highlighting, and diff-state crate. |
| `keryx` | Herald. HTTP, SSE, URL, and API error helper crate. |
| `bathron` | Pedestal. OS services for notifications, dialogs, settings, updates, and logging. |
| `parodos` | Stage entrance. Terminal UI substrate crate. |
| `dokimasia` | Examination. Design-token and standards linter crate. |
| consumer-pull | Rule that new public helpers land only after real consumer pressure is documented. |
| design tokens | Fleet CSS and UI vocabulary owned by kanon standards and consumed by skeue and dokimasia. |
| platform parity | Evidence that Linux, macOS, and Windows runtime behavior has been tested for an OS-hook surface. |
